# 基于STM32F103的环境监测系统
> 个人嵌入式项目 | 温湿度/空气质量/PM2.5多参数采集与报警终端

## 项目简介
设计并实现了一套**多路环境参数监测系统**，完成温湿度、空气质量、PM2.5浓度的实时采集与OLED显示。通过按键自定义报警阈值，超限自动触发LED、蜂鸣器与继电器联动报警。采用软件滤波优化传感器数据，针对性解决了MQ135采集异常问题，系统稳定可靠，可用于室内环境监测场景。

## 硬件平台
| 模块 | 型号/说明 |
|------|-----------|
| 主控 | STM32F103C8T6 最小系统板 |
| 传感器 | DHT11温湿度传感器、MQ135空气质量传感器、PM2.5粉尘传感器 |
| 显示 | 0.96寸 I2C OLED 显示屏 |
| 报警模块 | LED指示灯、有源蜂鸣器、5V继电器模块 |
| 开发环境 | Keil5 MDK、STM32标准库 |

## 核心功能
1.  **多传感器数据采集**：实时读取温湿度、空气质量、PM2.5浓度数据
2.  **可视化显示**：OLED屏幕分屏显示所有参数，界面清晰直观
3.  **自定义报警阈值**：通过按键设置温度、湿度、空气质量、PM2.5的报警上限
4.  **联动报警机制**：任意参数超限时，自动触发LED亮灯、蜂鸣器响铃、继电器动作
5.  **数据优化处理**：采用均值滤波+数据限幅，解决了MQ135传感器采集满量程异常的问题，提升数据稳定性

## 项目亮点
- 模块化编程设计，驱动代码与业务逻辑分离，结构清晰，便于后续维护与扩展
- 针对性解决了MQ135传感器采集数据异常的问题，优化了数据稳定性
- 实现了完整的闭环监测系统：数据采集→处理→显示→报警，功能完整可复现
- 所有参数可通过按键自定义配置，无需修改代码即可适配不同场景需求

## 使用说明
1.  将Keil工程文件下载到本地，使用Keil5打开工程文件（`.uvprojx`）
2.  编译代码并烧录到STM32F103开发板
3.  按接线图连接所有传感器与外设模块
4.  上电后OLED将显示实时环境数据，按下KEY0可进入报警阈值设置模式
